# Prompt Sửa Lỗi

```text
Vai trò: bạn là software engineer chịu trách nhiệm sửa lỗi.
Mục tiêu: sửa lỗi sau: ...
Bối cảnh:
- File/module liên quan: ...
- Hành vi hiện tại: ...
- Hành vi mong muốn: ...
- Bước tái hiện: ...
- Error/log: ...
Đầu ra mong muốn:
- Root cause
- Cách sửa ít rủi ro nhất
- Code fix
- Giải thích ngắn vì sao đúng
- Test cases cần thêm
- Regression risk nếu có
Ràng buộc:
- Ưu tiên sửa nhỏ, đúng trọng tâm
- Không refactor lan rộng nếu không cần
```
